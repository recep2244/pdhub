**Edit 1 — Define `_PIPELINE_MODES` at the top of Tab 1 (before the expander).**

At the very start of `with tabs[1]:` (around line 262, before the `with st.expander(...)` call), insert:

```python
_PIPELINE_MODES = {
    "LLM-guided (recommended)":     "llm",
    "Step-only (fast, no LLM)":     "step",
    "Antibody / Nanobody Design":   "nanobody_llm",
    "Binding Affinity Analysis":    "binding_affinity",
}
_cur_pm = st.session_state.get("p_mode", "LLM-guided (recommended)")
mode_str = _PIPELINE_MODES.get(_cur_pm, "llm")
```

This makes `mode_str` available to the expander (which is rendered before the selectbox widget in source order). On first page load, `p_mode` is not yet in `session_state`, so it defaults to `"llm"`. On re-runs after the user selects a mode, `st.session_state["p_mode"]` holds the selected label string.

**Edit 2 — Update the pipeline expander to use `mode_str`.**

Find this line inside the `with st.expander(...)` block (approximately line 266):
```python
st.markdown(_pipeline_table_markdown("llm"))
```

Replace with:
```python

```

**Edit 3 — Replace the pipeline mode selectbox and `use_llm` derivation in the Pipeline Settings column.**

Find this block (approximately lines 338-343):
```python
pm = st.selectbox("Pipeline mode", [
    "LLM-guided (recommended)",
    "Step-only (fast, no LLM)",
], key="p_mode")

```

Replace with:
```python
pm = st.selectbox("Pipeline mode", list(_PIPELINE_MODES.keys()), key="p_mode")

```

Note: `_PIPELINE_MODES` is already defined at the top of `with tabs[1]:` (Edit 1). The `mode_str` re-binding here picks up the actual widget value after the widget renders.
